Descripción de la Actividad:











Desarrolla un proyecto integrador que permita aplicar los conocimientos adquiridos sobre análisis de datos y técnicas de aprendizaje automático. El proyecto se divide en tres fases principales:











Selección o Creación del Conjunto de Datos (10 %):





Crear tu propio conjunto de datos 10%





Seleccionar uno disponible en repositorios abiertos especializados en Machine Learning (por ejemplo, Kaggle, UCI Machine Learning Repository, etc.). La elección debe considerar que el conjunto de datos tenga variables relevantes y suficientes registros para permitir un análisis significativo. 5%





Análisis Exploratorio y Preparación de los Datos (30%):





Aplica herramientas de estadística descriptiva para explorar en profundidad el conjunto de datos. Debes:





Describir cada una de las variables (tipos de datos, distribución, valores atípicos, etc.).





Identificar y tratar datos nulos o inconsistentes.





Realizar limpieza de datos si es necesario, cuidando la correcta transformación y codificación de las variables.





Aplicación de Técnicas de Machine Learning (60%):





Aplica diferentes técnicas de aprendizaje automático al conjunto de datos, considerando tanto modelos supervisados como no supervisados, de acuerdo con la naturaleza del problema (clasificación, regresión o clustering). Esta parte es el eje central del proyecto y debe desarrollarse con especial profundidad. 





























Las tareas específicas incluyen:











Selección y Justificación del Modelo:





Elegir al menos dos algoritmos distintos (por ejemplo: regresión logística, árbol de decisión, SVM, k-NN, redes neuronales, clustering k-means, etc.) y justificar su elección con base en el tipo de datos y la tarea a resolver.





Entrenamiento del Modelo:





Realizar el proceso completo de entrenamiento con los datos preparados. Se deben ejecutar varias corridas del modelo (al menos tres) con diferentes particiones o semillas aleatorias para validar la estabilidad de los resultados.





Evaluación del Modelo:





Utilizar métricas adecuadas al tipo de modelo aplicado. Por ejemplo:





Para clasificación: accuracy, matriz de confusión, F1-score.





Para regresión: MAE, RMSE, R².





Para clustering: silueta, inercia, etc.





Comparación de Resultados:





Presentar una tabla comparativa de los modelos utilizados y sus respectivas métricas. Justificar cuál modelo es más adecuado para el problema abordado, con base en los resultados obtenidos y su coherencia con el contexto del conjunto de datos.





Conclusiones Técnicas:





Redactar un apartado donde se analicen los hallazgos obtenidos a partir del modelo (o modelos) seleccionados, incluyendo limitaciones, posibles mejoras y recomendaciones para futuras versiones del análisis.
